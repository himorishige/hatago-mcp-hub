# Configuration Reference

Hatago uses a JSON config file (`hatago.config.json`).

## Basic Structure

```json
{
  "$schema": "https://raw.githubusercontent.com/himorishige/hatago-hub/main/schemas/config.schema.json",
  "version": 1,
  "logLevel": "info",
  "mcpServers": {}
}
```

## Fields

### version
- Type: `number`
- Required

### logLevel
- Type: `"debug" | "info" | "warn" | "error"`
- Default: `"info"`

### mcpServers
- Type: `object`
- Required: Define local/remote servers

## Local Server

```json
{
  "server-id": {
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-filesystem", "."],
    "env": { "KEY": "value" },
    "cwd": "./path",
    "disabled": false,
    "tags": ["dev", "local"]
  }
}
```

## Remote Server

```json
{
  "server-id": {
    "url": "https://api.example.com/mcp",
    "type": "http",
    "headers": { "Authorization": "Bearer ${TOKEN}" },
    "disabled": false,
    "tags": ["production"]
  }
}
```

## Env Expansion

- `${VAR}` → expand or error if undefined
- `${VAR:-default}` → use default if undefined

## Tag Filtering

```bash
hatago serve --tags dev,test
```
