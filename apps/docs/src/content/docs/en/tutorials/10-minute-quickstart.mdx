Hatago MCP Hub lets you manage multiple MCP servers from one place. Follow this guide to get to a working setup quickly.

## ✅ Prerequisites (1 min)

<Steps>
1. Check Node.js version
   ```bash
   node --version  # v20.0.0+
   ```

2. Pick your AI client
   - Claude Code (recommended)
   - Codex CLI
   - Cursor / Windsurf
</Steps>

## 🚀 Install (2 min)

Use `npx` for zero‑install usage:

```bash
# Create config interactively
npx @himorishige/hatago-mcp-hub init
```

## ⚙️ Create Config (2 min)

The `init` command walks you through an interactive setup.

<Steps>
1. Select mode
   ```
   ? Select mode:
   ❯ stdio (for Claude Code)
     http (for debugging)
   ```

2. Pick sample servers (e.g. filesystem)

3. Confirm files were created
   ```
   ✅ Created hatago.config.json
   ✅ Created .mcp.json (for Claude Code)
   ```
</Steps>

<Aside type="tip" title="Minimal config">
In a hurry? Save this as `hatago.config.json`:

```json
{
  "version": 1,
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "."]
    }
  }
}
```
</Aside>

## 🔌 Connect with Claude Code (2 min)

### If auto‑generated
`.mcp.json` is already created by `init`. Restart Claude Code.

### Manual setup

```json
{
  "mcpServers": {
    "hatago": {
      "command": "npx",
      "args": [
        "@himorishige/hatago-mcp-hub",
        "serve",
        "--stdio",
        "--config",
        "./hatago.config.json"
      ]
    }
  }
}
```

## ✅ Verify (2 min)

<Steps>
1. Open a new chat in Claude Code
2. List available tools (e.g. type `/tools`)
3. Try the filesystem tool ("list current directory")
4. Read internal resource: `hatago://servers` (via your client's resource panel or a JSON‑RPC call to `resources/read`)
</Steps>

    ```bash
    npx @himorishige/hatago-mcp-hub serve --http
    ```
    Then open http://localhost:3535/health
  </TabItem>

    ```bash
    npx @himorishige/hatago-mcp-hub serve --verbose
    ```

  </TabItem>
