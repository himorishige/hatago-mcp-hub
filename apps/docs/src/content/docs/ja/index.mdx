## 🏮 Hatagoとは

**Hatago (旅籠)** は江戸時代の旅人に宿を提供した日本の伝統的な宿です。現代のAIツールとMCPサーバーをつなぐ中継点として、Hatago MCP Hubは複数のMCPサーバーへの統合アクセスを提供します。

## ✨ 主な特徴

<CardGrid stagger>
  <Card title="🎯 シンプル＆軽量" icon="rocket">
    ゼロ設定で開始可能。既存プロジェクトに影響を与えない設計。
  </Card>
  <Card title="🔌 豊富な接続性" icon="puzzle">
    STDIO、HTTP、SSE、WebSocket対応。リモートMCPプロキシ機能付き。
  </Card>
  <Card title="🔥 ホットリロード" icon="refresh">
    設定ファイルの変更を自動検出。再起動不要で動的更新。
  </Card>
  <Card title="📊 進捗通知" icon="information">
    子サーバーからの通知を透過的に転送。長時間実行操作もサポート。
  </Card>
</CardGrid>

## 🚀 30秒で始める

```bash
# 設定の初期化
npx @himorishige/hatago-mcp-hub init

# サーバーの起動（Claude Code用）
npx @himorishige/hatago-mcp-hub serve --stdio --config ./hatago.config.json

# またはHTTPモード（デバッグ用）
npx @himorishige/hatago-mcp-hub serve --http
```

## 🎯 ユースケース

### 個人開発者向け

- 複数のMCPサーバーを一元管理
- 環境ごとの設定切り替え
- ローカル・リモートサーバーの混在利用

### チーム開発向け

- 共有設定ファイルによる環境統一
- タグベースのサーバーグループ管理
- CI/CDパイプラインとの統合

### エンタープライズ向け

- セキュアなプロキシ経由接続
- 監視・ログ収集との連携
- カスタムMCPサーバーの統合
