Hatago MCP Hubを使って、複数のMCPサーバーを10分で統合管理できるようにします。このガイドに従えば、すぐに動作確認まで完了します。

## 📋 前提条件の確認（1分）

<Steps>
1. **Node.jsのバージョン確認**
   ```bash
   node --version  # v20.0.0以上が必要
   ```

2. **使用するAIツールの確認**
   - Claude Code（推奨）
   - Codex CLI
   - Cursor / Windsurf
  </Steps>

<Aside type="tip">
  Claude Codeをまだインストールしていない場合は、[claude.ai/code](https://claude.ai/code)
  からダウンロードできます。
</Aside>

## 🚀 インストール（2分）

最も簡単な方法は`npx`を使うことです。インストール不要ですぐに使えます：

```bash
# 設定ファイルの初期化（対話型）
npx @himorishige/hatago-mcp-hub init
```

<Aside type="note">
  グローバルインストールを好む場合は： ```bash npm install -g @himorishige/hatago-mcp-hub hatago
  init ```
</Aside>

## ⚙️ 設定作成（2分）

`init`コマンドを実行すると、対話型で設定を作成できます：

<Steps>
1. **モードを選択**
   ```
   ? Select mode: 
   ❯ stdio (for Claude Code)
     http (for debugging)
   ```
   Claude Codeなら`stdio`を選びます。

2. **サンプルサーバーを選択**

   ```
   ? Select sample servers: (Space to select, Enter to confirm)
   ❯ ◉ filesystem - Browse and read files
     ◯ github - GitHub API access
     ◯ memory - Persistent memory storage
   ```

   試したいサーバーをスペースキーで選択します。

3. **設定ファイルの作成確認**
   ```
   ✅ Created hatago.config.json
   ✅ Created .mcp.json (for Claude Code)
   ```
</Steps>

<Aside type="tip" title="最小設定例">
急いでいる場合は、以下の最小設定を`hatago.config.json`として保存：

```json
{
  "version": 1,
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "."]
    }
  }
}
```

</Aside>

## 🔌 Claude Codeとの接続（2分）

### 自動設定の場合

`init`コマンドで`.mcp.json`が自動生成されています。Claude Codeを再起動するだけで接続完了です。

### 手動設定の場合

プロジェクトルートに`.mcp.json`を作成：

```json
{
  "mcpServers": {
    "hatago": {
      "command": "npx",
      "args": [
        "@himorishige/hatago-mcp-hub",
        "serve",
        "--stdio",
        "--config",
        "./hatago.config.json"
      ]
    }
  }
}
```

<Aside type="caution">
  Claude Codeの再起動が必要です。VS Codeメニューから「Developer: Reload Window」を実行してください。
</Aside>

## ✅ 動作確認（2分）

<Steps>
1. **Claude Codeで新しいチャットを開く**

2. **利用可能なツールを確認**

   ```
   利用可能なツールを表示して
   ```

   または

   ```
   /tools
   ```

3. **ファイルシステムツールをテスト**

   ```
   現在のディレクトリのファイル一覧を表示して
   ```

4. **内部リソースを確認**
   ```
   hatago://servers を読み取る（クライアントのリソース機能、または JSON-RPC `resources/read`）
   ```
   接続されているサーバー一覧が JSON で表示される。
</Steps>
